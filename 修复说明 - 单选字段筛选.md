# 🔧 修复：单选字段筛选条件

**问题**: 筛选条件使用了错误的值格式  
**修复**: 使用 option key 而不是 value  
**日期**: 2026-03-26

---

## 🐛 问题分析

### HAP 单选字段的数据格式

```json
"69b5dc145c2066509f21fd27": [
  {"key":"8e60a304-0bae-4a26-8c4a-2e01946c8960","value":"是"}
]
```

### 错误的筛选条件

```javascript
// ❌ 错误：使用 value
filter: {
  field: '69b5dc145c2066509f21fd27',
  operator: 'eq',
  value: ['是']  // ← 错误！
}
```

### 正确的筛选条件

```javascript
// ✅ 正确：使用 option key
filter: {
  field: '69b5dc145c2066509f21fd27',
  operator: 'eq',
  value: ['8e60a304-0bae-4a26-8c4a-2e01946c8960']  // ← option key
}
```

---

## ✅ 修复内容

### utils/api.js - getBanners()

**修改前**:

**修改后**:

value: ['8e60a304-0bae-4a26-8c4a-2e01946c8960']  // ✅ option key

## 📝 知识点

### HAP V3 API 单选字段筛选

**必须使用 option key**，不是显示值！

```javascript
// 文本字段 - 使用实际值
field: '名称',
value: ['广告 1']

// 单选字段 - 使用 option key
field: '是否启用',
value: ['8e60a304-0bae-4a26-8c4a-2e01946c8960']

// 多选字段 - 使用 option key 数组
field: '标签',
value: ['key1', 'key2']
```
