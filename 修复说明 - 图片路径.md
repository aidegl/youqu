# 🔧 修复：图片路径获取

**问题**: 附件字段需要解析 JSON 并取 `large_thumbnail_full_path`  
**修复**: 添加字符串转 JSON 逻辑  
**日期**: 2026-03-26 18:28

---

## 📋 附件字段数据结构

HAP 的附件字段可能返回两种格式：

### 格式 1: 字符串（需要解析 JSON）
```json
"69c4eb1c867350d552faf075": [
  "[{\"name\":\"image.png\",\"large_thumbnail_full_path\":\"https://...\"}]"
]
```

### 格式 2: 对象数组（直接使用）
```json
"69c4eb1c867350d552faf075": [
  {
    "fileName": "image.png",
    "large_thumbnail_full_path": "https://...",
    "downloadUrl": "https://..."
  }
]
```

---

## ✅ 修复逻辑

```javascript
const imageField = row['69c4eb1c867350d552faf075'];
let imageUrl = '';

if (imageField && Array.isArray(imageField) && imageField.length > 0) {
  const firstImage = imageField[0];
  
  // 如果是字符串，需要解析 JSON
  if (typeof firstImage === 'string') {
    try {
      const parsed = JSON.parse(firstImage);
      imageUrl = parsed[0]?.large_thumbnail_full_path || parsed?.large_thumbnail_full_path || '';
    } catch (e) {
      imageUrl = firstImage;
    }
  } else if (typeof firstImage === 'object') {
    // 如果已经是对象，直接取 large_thumbnail_full_path
    imageUrl = firstImage.large_thumbnail_full_path || firstImage.downloadUrl || '';
  }
}
```

---

## 🎯 优先级

图片 URL 获取优先级：
1. `large_thumbnail_full_path` ← 首选
2. `downloadUrl` ← 备选
3. `url` ← 备选
