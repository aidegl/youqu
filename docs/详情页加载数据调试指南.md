# 详情页加载数据调试指南

## 问题背景

在小程序开发中，从列表页跳转详情页是常见场景。本文记录了一次完整的调试过程和方法。

---

## 数据流程

### 1. 列表页跳转

**文件**: `pages/index/index.wxml`

```xml
<view 
  class="post-card" 
  wx:for="{{posts}}" 
  wx:key="id"
  data-id="{{item.id}}"
  bindtap="goToDetail"
>
```

**关键点**:
- 使用 `data-id` 传递帖子 ID
- `wx:key="id"` 确保列表渲染正确

**文件**: `pages/index/index.js`

```javascript
goToDetail(e) {
  const postId = e.currentTarget.dataset.id;
  console.log('[goToDetail] dataset:', e.currentTarget.dataset);
  console.log('[goToDetail] postId:', postId);
  
  wx.navigateTo({
    url: `/pages/post/post?id=${postId}`
  });
}
```

---

### 2. 详情页接收参数

**文件**: `pages/post/post.js`

```javascript
Page({
  data: {
    postId: '',
    post: null,
    debugInfo: null  // 调试信息
  },

  onLoad(options) {
    if (options.id) {
      this.setData({ postId: options.id });
      this.loadData();
    }
  }
});
```

**关键点**:
- `onLoad(options)` 接收 URL 参数
- `options.id` 对应 URL 中的 `?id=xxx`

### 3. API 调用获取数据

**文件**: `utils/api.js`

```javascript
async function getPostDetail(postId) {
  console.log('===== [getPostDetail] 开始 =====');
  console.log('帖子 ID:', postId);
  
  // 使用列表 API 通过 rowid 筛选获取记录
  const result = await getRows(WORKSHEET_ID.posts, {
    filter: {
      type: 'group',
      logic: 'AND',
      children: [{ 
        type: 'condition', 
        field: 'rowid', 
        operator: 'eq', 
        value: [postId]  // 注意：用数组格式
      }]
    },
    pageIndex: 1,
    pageSize: 1
  });
  
  console.log('API 响应 success:', result.success);
  console.log('API 响应 error_msg:', result.error_msg);
  console.log('rows 数量:', result.data?.rows?.length || 0);
  
  if (result.success && result.data?.rows && result.data.rows.length > 0) {
    const row = result.data.rows[0];
    console.log('获取到记录 ID:', row.rowid || row.rowId);
    
    return { success: true, data: formatPost(row) };
  }
  
  return { success: false, data: null };
}
```

**关键点**:
- 使用 `getRows` (列表查询) 而不是 `getRow` (单条查询)
- 筛选条件 `value` 必须用**数组格式** `[postId]`
- HAP V3 API 的筛选器语法

### 4. 数据格式化

**文件**: `utils/api.js`

```javascript
function formatPost(row) {
  // 获取作者昵称（直接在 row 层级）
  let authorNickname = '未知';
  if (row.zznc && typeof row.zznc === 'string') {
    authorNickname = row.zznc;
  }
  
  // 获取作者头像（直接在 row 层级，和封面图同样方式）
  let authorAvatar = 'https://fp1.mingdaoyun.cn/customIcon/0_lego.svg';
  if (row.zztx && Array.isArray(row.zztx) && row.zztx.length > 0) {
    const avatarImg = row.zztx[0];
    if (typeof avatarImg === 'object') {
      authorAvatar = avatarImg.downloadUrl || avatarImg.large_thumbnail_full_path || avatarImg.url || '';
    }
  }
  
  // 获取帖子 ID（兼容 rowid 和 rowId）
  const id = row.rowid || row.rowId || '';
  
  return {
    id: id,
    title: row.title || '',
    content: row.content || '',
    author: {
      id: row.author_id?.[0]?.sid || '',
      nickname: authorNickname,
      avatar: authorAvatar
    },
    // ... 其他字段
  };
}
```

**关键点**:
- HAP 返回的数据中，自定义字段（如 `zznc`、`zztx`）直接在 `row` 层级
- 关联字段（如 `author_id`）返回的是数组
- 图片字段是对象数组，需要取 `downloadUrl` 或 `large_thumbnail_full_path`

## 常见问题排查

### 问题 1: postId 是 undefined

**可能原因**:
1. 列表页 `data-id` 绑定错误
2. `formatPost` 返回的 `id` 为空

**排查步骤**:
1. 检查列表页 WXML：`data-id="{{item.id}}"`
2. 检查 `formatPost` 函数：确保 `row.rowid` 或 `row.rowId` 有值
3. 在 `goToDetail` 函数打印 `e.currentTarget.dataset`

---

### 问题 2: API 返回"数据不存在"

**可能原因**:
1. 帖子 ID 格式错误
2. 筛选条件 `value` 格式错误

**解决方案**:
```javascript
// ❌ 错误：value 是字符串
value: postId

// ✅ 正确：value 是数组
value: [postId]
```

---

### 问题 3: 作者头像/昵称显示异常

**可能原因**:
1. 字段层级搞错（在 `row` 层级还是在 `author_id` 里）
2. 图片字段解析方式错误

**排查步骤**:
1. 打印原始 `row` 数据：`console.log(JSON.stringify(row, null, 2))`
2. 确认字段位置：`row.zznc` vs `row.author_id[0].zznc`
3. 图片字段取第一个元素的 `downloadUrl`

---

## 调试检查清单

在排查详情页加载问题时，按顺序检查：

- [ ] 列表页 `data-id` 正确绑定
- [ ] `goToDetail` 函数能获取到 `postId`
- [ ] URL 跳转格式正确：`/pages/post/post?id=xxx`
- [ ] 详情页 `onLoad(options)` 能接收到 `options.id`
- [ ] API 请求的 `filter` 格式正确
- [ ] API 响应 `success: true`
- [ ] `formatPost` 返回的数据完整
- [ ] 页面 `setData` 后 `post` 对象有值

---

## 总结

调试详情页加载数据的核心方法：

1. **链路追踪**: 从点击 → 跳转 → 接收 → 请求 → 格式化 → 渲染，每个环节打日志
2. **页面调试框**: 把关键信息显示在页面上，避免反复复制日志
3. **原始数据打印**: 遇到字段取值问题，先打印原始数据看结构
4. **对比法**: 对比列表页和详情页的数据格式差异
